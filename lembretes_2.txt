		- Após a aprovação o documento se chamará PI

		- o card Pendente de informações deve mostrar a quantidade de Ocs que estão no sistema como rascunho ( pois ainda não tem todos os dados completos para enviar para aprovação)
		- O card prazo estourado deve fazer a contagem de OCs que estão estourando o prazo de atendimento (preenchimento de todos os dados, cadastrando-os para criar OCs) 
		- O Tempo de cada etapa para saber o estouro de tempo deve ser configurado em um submenu novo que deve ser adicionado ao módulo no menu lateral "executivo" 
			- nessa tela pode-se configurar quantos dias devem ser a etapa de criação da OC (desde o dia que ela foi iniciada, até o envio para a requisição de aprovação)		- Configurar quantos dias uma OC pode aguardar aprovação para virar uma PO
